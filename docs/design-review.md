# DoubleDone design review

*Generated 2026-06-26 by an 8-lens code-level design audit (colour, type, spacing, duplication, responsive, accessibility, motion, brand). Every finding was adversarially re-checked against the code by a second agent, so intentional choices were filtered out. A living polish backlog: burn down Tier 1 first, re-run the audit when it is clear.*

---

## Read

DoubleDone's design system is in good shape on the things that are easy to see and quietly broken on the things that only show up in dark mode or under a screen reader. The light-mode surface reads coherent, the token file already carries spacing, radius, motion and layout, and the reduced-motion contract is honoured almost everywhere. The drift is concentrated in three places: a missing per-scheme `onAccent`/`onDone` colour that lets every white CTA label and completion tick fall to 2.1 to 2.8:1 in dark mode (a real WCAG failure on the most-used buttons, invisible in the light preview), a missing type scale that has let title and label sizes splinter into ~18 hand-tuned literals, and a pile of copy-pasted primitives (the modal card, the accent button, the scrim, the pressed state) that have each already drifted because no component owns them. The three highest-leverage moves, in order: add the `onAccent`/`onDone`/`scrim` colour tokens and fix dark-mode contrast, add the named type scale, and extract the four shared components (`ModalCard`, `PrimaryButton`, `Chip`/`Segmented`, `BackLink`). Do those and most of the punch-list below collapses on its own.

## Tokens and components to introduce

Ranked by leverage. Each kills a whole class of drift at once.

- **`typeScale` token in theme.ts** (named display / title / heading / subheading / body / bodyStrong / label / caption, each bundling size + lineHeight + family + weight + letterSpacing, built inside `buildTheme` so `t.scale` is baked in). Kills the entire typography class: page-title 42-vs-34, modal-title 21-vs-22 and 24-vs-26, the 8-way eyebrow/label mess, the 600-vs-700 CTA-weight drift, and the Today-title 700 that Newsreader cannot render. Touches ~30 sites across every screen and component.
- **`PrimaryButton` component** (accent bg, `onAccent` label, one radius, one label size, one pressed/disabled state). Kills the most-pressed-surface drift: 3 corner radii, 2 label sizes, 2 pressed opacities reinvented ~12 times. Touches ~12 sites.
- **`onAccent` + `onDone` colour tokens** (light keeps `#FFFFFF`, dark uses warm ink ~`#2B2722`). Kills the dark-mode contrast failure on every accent CTA label, selected calendar day, step tick and completion tick. Touches ~18 sites for `onAccent`, 2 for `onDone`.
- **`ModalCard` scaffold** (Modal + scrim Pressable + centred tap-absorbing card, width from a token). Kills the rebuilt-overlay class and folds the eight in-file Today modals onto one surface. Touches 4 hand-built sites plus ~13 modals.
- **`scrim` colour token** (warm-ink wash, both schemes). Kills the 5-way duplicated `rgba(43,39,34,0.45)` literal and the drifted cold-black Rooms scrim. Touches 6 sites.
- **`pressed` style token** (one canonical opacity, 0.7, with PremiumButton's gradient 0.9 as the one documented exception). Kills the 6-value pressed-opacity drift. Touches ~15 sites.
- **`Chip` + `Segmented` components** (one base shape, one "on" treatment, role + label + selected by construction). Kills the solid-vs-soft active-chip split, the Settings/Breakdown segmented duplication, and carries the a11y label/selected state so pills can't ship label-less. Touches ~8 sites.
- **`BackLink` component** (chevron + accent text + `canGoBack` fallback + a11y). Kills the 6-way copy-paste and forces Lookback onto the safe navigation fallback and one text style. Touches 6 sites.
- **`layout.modalWidth` + `layout.cardMediaWidth` + `layout.maxCalendarWidth` tokens**. Kills the 420-vs-440-vs-360 dialog-width drift and the host-dependent DatePicker/Lookback calendar sizing. Touches ~11 sites.
- **`control` size family + border-width family in theme.ts** (one checkbox size, named hair/thin/medium/thick borders). Kills the 26-vs-24-vs-22 checkbox drift and the 1.5-vs-2 selected-emphasis drift. Touches ~16 sites.
- **`cardShadow` helper / `<Card>`** (scheme-aware elevation recipe, defined once). Kills the byte-identical boxShadow duplicated in TaskRow and Routines. Touches 2 sites.
- **`Mark`/`Glyph` SVG component + hitSlop helper**. Kills emoji-as-icon (so every mark tints via `t.colors` and dark-mode-adapts) and the scattered missing-hitSlop class in one pass. Touches ~7 emoji/glyph sites plus ~6 under-target controls.

## Tier 1 (must)

Breaks consistency, dark mode, or the calm never-shame brand.

- **White CTA labels and selected-state glyphs fail contrast in dark mode.** `theme.ts:58` (dark accent `#C68BA0`), plus the `#FFFFFF` labels on accent backgrounds at index.tsx:2070, 2142, 2177, 2203; chart.tsx:258, 275; premium.tsx:264; settings.tsx:617; welcome.tsx:380; lookback.tsx:537, 440; DatePicker.tsx:106; TaskRow.tsx:339; BrainDump.tsx:601, 524; BreakdownQuestions.tsx:206; BreakdownReview.tsx:212. White on `#C68BA0` is 2.76:1, failing AA and AA-large. Add a per-scheme `onAccent` token (dark = warm ink ~`#2B2722`, verified 5.37:1) and route every `#FFFFFF`-on-accent through it, verifying each at the AA threshold in dark mode. (judgment-call)
- **White completion tick is 2.16:1 on the sage done fill in dark mode.** `theme.ts:60` (dark done `#9DB98A`), TaskRow.tsx:339 (tick), welcome.tsx:373 (checkmark). The most emotionally loaded mark in a never-shame app reads as a pale smudge in dark mode. Add an `onDone` token (light keeps `#FFFFFF`, dark uses warm ink, verified 6.88:1) and point both ticks at it. The light-mode white tick is intentional and stays. (judgment-call)
- **routines.tsx is the only full-screen route with no maxWidth cap.** routines.tsx:242 (no maxWidth/centring), routines.tsx:6 (never imports `layout`). On wide desktop every routine card and the add-form stretch edge-to-edge while the user arrived from a centred Today/Lookback, breaking the calm centred-column the token exists to guarantee. Import `layout`, add `maxWidth: layout.maxContentWidth, width: '100%', alignSelf: 'center'`, then bake this into a shared `<Screen>` wrapper so no future route can ship uncapped. (mechanical)
- **RepeatingDrawer ignores reduced-motion entirely.** RepeatingDrawer.tsx:29-35. Every other animated surface collapses to end-state under `useReducedMotion`; this one always runs the slide-and-fade, a broken accessibility promise for the exact motion-averse audience the app is built for. Import `useReducedMotion`, snap to end-state when reduced, mirroring index.tsx closeRise. (mechanical)
- **RepeatingDrawer toggle rows announce as "button", not "checkbox".** RepeatingDrawer.tsx:62-69. The rows already pass `accessibilityState={{checked: done}}` but use `role='button'`, so toggling a daily task reads with no state change, inconsistent with TaskRow's correct `role='checkbox'`. One-word change to `role='checkbox'`. (mechanical)
- **Emoji glyphs used as icons break the editorial mark vocabulary and lean alarm.** TaskRow.tsx:210, 268 (`🔔`), BrainDump.tsx:258 (`🎤 Speak`), 270 (`📷 Scan`). The emoji render in raster multicolour, ignore `t.colors`, never tint mauve, never adapt to dark mode, and the inline bell beside a task title leans toward the notification/alarm cue the never-alarm vocabulary forbids. Replace with the typographic/SVG mark family tinted via `t.colors`: drop the bell for the accent-coloured time alone, use text or thin SVG for Speak/Scan. (judgment-call)
- **Rooms sheet scrim is cool pure-black, against the warm-not-black rule.** RoomsSheet.tsx:73 (`rgba(0,0,0,0.28)`). Every other overlay uses the warm ink `rgba(43,39,34,0.45)`, and this is the scrim behind the app's main navigation door, hit constantly. Both the brand rule ("never a terminal black") and consistency point the same way. Route it through the shared `scrim` token. (mechanical, folded with the scrim-duplication item below)

## Tier 2 (should)

Real drift or duplication worth fixing.

- **No named type scale: font sizes are inline literals everywhere.** theme.ts:126 (fonts export families only), with literals spanning ~18 steps (11 to 42) across index.tsx, lookback.tsx, premium.tsx, settings.tsx, welcome.tsx, TaskRow.tsx, BreakdownReview.tsx and more. The brand's typographic voice can't be tuned in one edit and each screen guesses a size. Add the `typeScale` token; the next four items all collapse once it exists. (judgment-call)
- **Three page-title sizes (42 / 34) for the same slot.** 42 on settings.tsx:503, privacy.tsx:104, premium.tsx:227, routines.tsx:244; 34 on index.tsx:1998, lookback.tsx:415, chart.tsx:234. Two screens side by side feel the seam. Decide deliberately whether Today/Lookback/Chart read lighter (name `display` vs `title`) or unify to one step. The 40 welcome wordmark stays a named brand exception. (judgment-call)
- **The uppercase eyebrow/label reinvented 8 times with mismatched size, weight, tracking.** index.tsx:2011, 2170; TaskRow.tsx:350; settings.tsx:505; routines.tsx:261; BreakdownReview.tsx:170; RoomsSheet.tsx:98; Bloom.tsx:112. Size 11/12/13, weight 600/700, letterSpacing 0.3/0.4/0.5/0.8/1 for the same quiet section eyebrow. Add one `label` entry to the type scale and spread it at all 8. (judgment-call)
- **Near-duplicate title sizes that should collapse: 21-vs-22 and 24-vs-26.** index.tsx:2125 (21) vs 2060/CameraCapture.tsx:208 (22); BreakdownReview.tsx:139, BreakdownQuestions.tsx:192, premium.tsx:231 (24) vs index.tsx:2095, 2195, RepeatingDrawer.tsx:102, sign-in.tsx:217 (26). Neither gap reads as intentional hierarchy. Collapse 21 into 22, pick one of 24/26 as the heading step. (judgment-call)
- **600-vs-700 weight drift on the same primary-CTA role, plus redundant weight literals.** welcome.tsx:380, PremiumButton.tsx:39, chart.tsx:258 (w700) vs sign-in.tsx:235, index.tsx:2107, BreakdownQuestions.tsx:242, CameraCapture.tsx:216 (w600); settings.tsx:516, 530 pair `bodyBold` with a redundant explicit weight. Let a `bodyStrong`/`button` type entry own the weight and drop the per-site literals. (judgment-call)
- **The Today title asks for weight 700, which Newsreader cannot render.** index.tsx:1999. theme.ts:124 states 600 is the heaviest loaded weight, so 700 clamps to 600 on native and renders heavier than every sibling page title on web. Set to 600; resolved once when titles route through the token. (mechanical)
- **The solid-accent primary CTA reimplemented ~12 times with drifted radius and label size.** premium.tsx:262/264, chart.tsx:248/258, sign-in.tsx:229/235, welcome.tsx:379/380, lookback.tsx:530/537, index.tsx:2141/2176/2202/2062, BrainDump.tsx:593/601, BreakdownQuestions.tsx:234/242, BreakdownReview.tsx:205/212, CameraCapture.tsx:210/216. Radius is `lg`/`md`/`pill`, label 16/17, pressed 0.8/0.85 across the app's most-pressed surface. Extract `<PrimaryButton>`, pick one radius and label size (`md`/17 most common), confirm the lookback pill and premium/chart `lg` as outliers or fold them in. (judgment-call)
- **The centred modal card scaffold is rebuilt in 4 files for ~13 modals.** index.tsx:2178/2186 (backs 8 modals), BrainDump.tsx:566/573, BreakdownQuestions.tsx:183/190, BreakdownReview.tsx:130/137. Same backdrop + tap-absorbing card, with width already split 420/440/360. Extract `<ModalCard visible onClose maxWidth?>`, drive scrim from the `scrim` token and width from a layout token. (judgment-call)
- **Modal scrim is a magic literal duplicated across 5 files (and already drifted in Rooms).** index.tsx:2180, BrainDump.tsx:568, BreakdownQuestions.tsx:185, RepeatingDrawer.tsx:88, BreakdownReview.tsx:132 (`rgba(43,39,34,0.45)`), plus RoomsSheet.tsx:73 (the cold-black drift in Tier 1). A future scrim tweak or a dark-specific opacity bump has to be made six times. Add the `scrim` token (consider a heavier dark opacity, since 0.45 ink over `#1B1917` dims far less than over paper) and point all six at it. (mechanical)
- **Centred-dialog widths split 420 / 440 / 360 with no token.** index.tsx:2185-2191 (420), index.tsx:2169 / BreakdownQuestions.tsx:190 / BreakdownReview.tsx:137 (440), BrainDump.tsx:578 / lookback.tsx:481, 511 (360). Moving from "Plan my day" to "Break it down" shifts the dialog 20px wider for nothing. Add `layout.modalWidth` (440 wins on count) and `layout.cardMediaWidth` (360). (judgment-call)
- **Completion checkbox is three sizes across screens (26 / 24 / 22).** TaskRow.tsx:318-326 (26), BreakdownReview.tsx:154-162 / chart.tsx:265-273 / RepeatingDrawer.tsx:109-117 (24), routines.tsx:275-283 / welcome.tsx:354 (22); the 34x34 tap square also recurs verbatim in BrainDump and lookback.tsx:436. Add a `control` size family, pick one checkbox size so the tick reads identically everywhere. The genuinely-shared round sage check (TaskRow + BreakdownReview only) can become `<CheckCircle done size?>`; chart's accent selection check, routines' square box and welcome's hollow bullet are different marks and stay. (judgment-call)
- **Selected-emphasis border weight is 1.5 on some screens, 2 on others.** TaskRow.tsx:290, 292, 322 (2) vs settings.tsx:529, chart.tsx:269, lookback.tsx:444, 514, welcome.tsx:347 (1.5). Border width is load-bearing (1 = divider, 1.5/2 = selected) but untokenised and inconsistent for "selected". Add a border-width family and reconcile selected to one named value. CameraCapture's 3px shutter ring is decorative and stays out. (judgment-call)
- **The segmented pill control is duplicated between Settings and BreakdownQuestions and has drifted.** settings.tsx:468-485/518-531 (active border 1.5, segText w700) vs BreakdownQuestions.tsx:116-134/208-220 (active border 1, segText w600), down to identical style names. Extract `<Segmented value options onChange>`; Settings keeps its label/hint wrapper. One border, one weight for the active pill. (mechanical)
- **The selectable chip has two different active treatments.** BrainDump.tsx:514-524 and BreakdownQuestions.tsx:196-206 fill the active chip solid mauve with white text; chart.tsx:253-256 uses the calmer `accentSoft`-tint-with-accent-text. A chip lights mauve-on-white in one flow and soft-tint in another. Extract `<Chip variant>`, default to the calmer `soft` (matches the segmented control), migrate the four sites. (judgment-call)
- **The "‹ Back" link is copy-pasted into 6 sub-screens, and Lookback's copy has diverged.** settings.tsx:190-197/501, chart.tsx:118-125/233, routines.tsx:113-120/243, premium.tsx:108-115/226, privacy.tsx:23-30/103, lookback.tsx:171-173/414. Lookback diverges on style (`inkSoft` body 16) and behaviour (bare `router.back()` with no `canGoBack` fallback, so a deep-linked Lookback can dead-end). Extract `<BackLink>`; migration forces Lookback onto the safe fallback. (judgment-call)
- **Pressed-state opacity has 6 values and no token.** TaskRow.tsx:307 / settings.tsx:532 (0.7), settings.tsx:400 (inline 0.85), RoomsSheet.tsx:91 (0.6), sign-in.tsx:236 / premium.tsx:265 / chart.tsx:259 / CameraCapture.tsx:264 / BrainDump.tsx:599 (0.8), welcome.tsx:382 / lookback.tsx:550 / index.tsx:2120 / BreakdownReview.tsx:213 / BreakdownQuestions.tsx:243 (0.85), PremiumButton.tsx:40 (0.9). The single most common gesture dims to five different opacities. Pick one (0.7) as a `pressed` token, keep PremiumButton's gradient 0.9 as the one documented exception. (judgment-call)
- **The premium gradient's honey end is a second warm accent on the protected Today screen.** theme.ts:73, PremiumButton.tsx:28, index.tsx:1266. The full mauve-to-honey gradient renders as the inline "Plan my day" button on Today, introducing a gold the single-accent Dusk palette never otherwise shows on the core screen. Keep the gradient for dedicated premium surfaces; give the inline-on-Today affordance a mauve-only fill with a small "Premium" tag, and document the rule beside `PREMIUM_GRADIENT`. (judgment-call)
- **Date chips and segments have no hitSlop across three screens, unlike BrainDump's.** BreakdownQuestions.tsx:69-92, 120-132; chart.tsx:149-159. `paddingVertical` of 8px puts them near 30px tall, under the 44pt floor, while BrainDump's identical chips add `hitSlop={{top:8,bottom:8}}`. Add the matching hitSlop. (mechanical)
- **BrainDump slice steppers and weekday circles have missing or partial hitSlop.** BrainDump.tsx:372-388 (slice steppers, none), 296-309 (weekday circles, top/bottom only), routines.tsx:198-208 (when-pills near 36px). The visually identical everyN steppers in the same file get `hitSlop={8}`. Add `hitSlop={8}`, extend the weekday slop to all four sides, raise the when-pill target. (mechanical)
- **TaskRow confirm-row inline actions have zero hitSlop and sit under 44pt.** TaskRow.tsx:102-115, 127-152. The densest set of unpadded inline actions in the app, ~20px tall at 15px text, where the select-bar and Settings actions get `hitSlop={6}`. Give each a hitSlop reaching ~44pt or a centred `minHeight:44`, verifying no overlap when the confirm row wraps on a narrow phone. (judgment-call)
- **Decorative row glyphs (↻, ★, 🔔) aren't folded into the row label and aren't hidden from a11y.** TaskRow.tsx:74 (rowLabel), 208-212, 266-271. The label names "pinned" and "big" but not the repeat or nudge state, so a recurring task and a reminder task read identically, and the glyph Text nodes may be double-announced as stray characters. Fold "repeating" and the nudge time into rowLabel and hide the glyphs from a11y. (judgment-call)
- **Modal scrims carry a dismiss label but no `role='button'`.** BreakdownQuestions.tsx:56, BreakdownReview.tsx:52, RepeatingDrawer.tsx:43. The canonical scrim at index.tsx:1552 pairs label with `role='button'`; these three announce a label and no role, reading as a giant typeless control. Add `role='button'` to match. (mechanical)

## Tier 3 (defer)

Minor or nice-to-have.

- **RepeatingDrawer hardcodes `useNativeDriver:false` against convention.** RepeatingDrawer.tsx:33. Only opacity and translateX animate, both native-driver-safe, and the app standard is `Platform.OS !== 'web'`; the `false` pushes the slide onto the JS thread on the exact Android class the app targets. Switch it, or document inline if some Android issue forced it. (mechanical)
- **Two animation durations are written as magic numbers that are already motion tokens.** index.tsx:289 (320 = `motion.gentle`), RepeatingDrawer.tsx:32 (200 = `motion.standard`). A later token change would silently skip them. Route both through the existing tokens. (mechanical)
- **closeRise and RepeatingDrawer drift from the easing convention.** index.tsx:290 (`Easing.out(Easing.cubic)`, sharper than the app's `Easing.ease` family), RepeatingDrawer.tsx:30-34 (no easing passed). Bring closeRise into the family or document the cubic, give the drawer an explicit `inOut`. Optionally promote the convention from a comment to named easing constants. (judgment-call)
- **RotatingPhrase cross-fade uses 500ms, a duration in no token tier.** RotatingPhrase.tsx:37, 44. Either pull to `motion.gentle` (320) or, if the slow ambient fade is intended, add a named `motion.crossfade` and reference it in both halves rather than a bare literal twice. (judgment-call)
- **RepeatingDrawer's Close button and task rows have no pressed feedback.** RepeatingDrawer.tsx:48, 62-65. Tapping a repeating task has no in-the-moment acknowledgement, only the eventual checkbox flip. Apply the shared `pressed` token; sequence after it exists. (mechanical)
- **Per-scheme card boxShadow recipe is copy-pasted byte-for-byte.** TaskRow.tsx:288, routines.tsx:269. Correctly theme-aware, just duplicated, so an elevation change must touch two places. Lift into a `cardShadow(t)` helper or `<Card>`. (mechanical)
- **The completion check circle is shared by TaskRow and BreakdownReview but split on geometry (26 vs 24).** TaskRow.tsx:318-327/339, BreakdownReview.tsx:154-164. The same round sage done-check at two diameters on the app's signature gesture. Extract `<CheckCircle done size?>`, standardise one diameter, keep the intentional white-on-sage tick inside. (judgment-call)
- **DatePicker day cells inflate with host card width.** DatePicker.tsx:88, 100, 102 (`flex:1, aspectRatio:1`). The same picker is visibly taller in the 440 and 420 host cards than in the 360 one. Give `wrap` an internal `maxWidth` (~340) and `alignSelf:'center'` via a `layout.maxCalendarWidth` token. (judgment-call)
- **Routines when-pills set selected state but carry no accessibilityLabel.** routines.tsx:198-208. Equivalent pills in BrainDump, BreakdownQuestions and chart all pass an explicit label; only the visible Text is announced here. Add the label (or fold into a shared chip primitive). (mechanical)
- **Exact-match raw spacing values written as numbers.** index.tsx:1990 (`paddingVertical:8` = `spacing.two`), RoomsSheet.tsx:95 and lookback.tsx:444 (`marginTop:2` = `spacing.half`). Pure token hygiene, zero visual change. Swap only the exact matches; leave off-scale nudges (13, gap:3) and sub-2px values alone. (mechanical)
- **Bloom splits its typography between StyleSheet and inline JSX.** Bloom.tsx:96-99 (family/size/lineHeight inline), 107-114 (StyleSheet holds only colour/style). Convert to a themed `makeStyles(t)` and reference the named type steps once they exist; the fixed celebration colours stay. (judgment-call)
- **Periwinkle does double duty for slice counts and the make-it-tiny eyebrow.** TaskRow.tsx:350, 353. `t.colors.repeat` is the recurrence mark but also colours a one-off task's "2 / 5" slice count, a mild semantic overload. Either widen the palette comment to "structured/multi-part" or move sliceCount and tinyEyebrow to `inkSoft`/`accent`. (judgment-call)

## Tier 4 (skip)

Noted but deliberately not doing.

- **The "Big" capitalised weight pill.** TaskRow.tsx:208, 266. It is the only capitalised single-word tag in an otherwise lowercase microcopy system, and softening to "a big one" matches the Lookback register. Skipped here as content polish, not design-system drift; worth a one-line copy tweak when TaskRow is next open, but it does not earn a token or component. (judgment-call)
- **Lookback month-grid blob density on wide desktop.** lookback.tsx:435-436. Re-verified: the cells are `flex:1` with no `aspectRatio`, so the 34px blobs do not inflate, they just gain horizontal air. Not the inflation originally claimed, only a mild proportion nicety. Cap it with the shared `maxCalendarWidth` token if you touch the grid anyway, otherwise leave it. (judgment-call)
- **Footer dock 700px breakpoint vs the 560 content cap.** index.tsx:110, 1295, 2022-2035. Between 561 and 699px the dock bar sits mid-page rather than edge-to-edge, purely cosmetic. Drive the threshold off the shared layout constant if convenient, but it is not worth a dedicated change. (judgment-call)

## Verified-and-dropped

The verifiers filtered out a fair number of auditor flags as intentional, so this list is the residue after that pass. Worth knowing what was deliberately left alone: the `t.scale` text-size multiplier and the sans/body/bodyBold family split are applied consistently and are correct (the new type scale must preserve the baked-in `t.scale`). The 40px welcome wordmark, the welcome 480 and privacy 640 max-widths are documented brand exceptions, not drift. The Rooms-pill glassy translucent fill and the welcome pagination dots branch per scheme correctly and are deliberate. Input min-heights (150 for the welcome dump, 88/90 for inline boxes) are per-context sizes, not a drifted single value. The native-only "Remind me" chip, the full-screen Focus and CameraCapture takeovers (which cap an inner block, not the page), the OS spinners and the static listening dot left ungated under reduced motion, the shared `PREMIUM_GRADIENT` constant and its differently-sized renderings, the BrainDump-local +/- stepper, and the CameraCapture 3px shutter ring are all intentional. Two were dropped as outright wrong: the weight gauge already exposes its load as readable text ("A full day"), and welcome.tsx has no sub-44 when-pills (its only inline controls already carry `hitSlop={10}`). The list above is what survived that filter.
